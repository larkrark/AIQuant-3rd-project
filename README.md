## 문서 수정 방법 (위키 방식)

1. GitHub 웹에서 md 파일 열기 → 연필 아이콘 → 수정 → Commit changes
2. 커밋하면 1~2분 내 위키에 자동 반영됩니다 (별도 빌드 없음)
3. docx 원본을 수정한 경우, 같은 이름의 .md 사본도 함께 갱신해 주세요 (md가 위키에 표시되는 판입니다)
4. 팀 결정이 걸린 내용은 반드시 [결정로그](01_운영문서/결정로그.md)에 기록합니다

## 새 문서 등록/생성 방법

### GitHub 웹에서 바로 만드는 경우

1. 저장소에서 문서를 넣을 폴더로 이동합니다.
   - 운영 기준 문서: `01_운영문서`
   - 회의록: `02_회의기록`
   - 조사자료: `03_조사자료`
   - 기획·방법론: `04_기획·방법론`
   - 참고·아카이브: `05_참고·아카이브`
2. **Add file → Create new file**을 누릅니다.
3. 파일명을 `폴더명/파일명.md` 형식으로 입력하거나, 현재 폴더 안에서 `파일명.md`로 입력합니다.
4. 본문을 작성한 뒤 **Commit changes**를 누릅니다.
5. 위키 왼쪽 메뉴에 노출해야 하는 문서는 [_sidebar.md](_sidebar.md)에 링크를 추가합니다.

### Word 문서를 등록하는 경우

1. 원본 `.docx`는 해당 폴더에 업로드합니다.
2. GitHub Pages에서 읽을 수 있도록 같은 이름의 `.md` 변환본도 함께 등록합니다.
   - 예: `260720_상세회의록_팀공유본.docx`
   - 예: `260720_상세회의록_팀공유본.md`
3. 위키에 보여줄 문서는 `.md` 파일을 [_sidebar.md](_sidebar.md)에 연결합니다.

### 등록 규칙

- 파일명 앞에는 가능하면 날짜를 `YYMMDD` 형식으로 붙입니다.
- 현행 기준 문서는 `01_운영문서`에 두고, 구버전·참고용 문서는 `05_참고·아카이브`로 이동합니다.
- 새 문서가 팀 결정, 규칙, Mandate, BM, 환율, 가중, 적용일에 영향을 주면 [결정로그](01_운영문서/결정로그.md)에 결정번호를 남깁니다.
