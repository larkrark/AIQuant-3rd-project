# 3차 프로젝트 브레인스토밍 — AI/ML 기반 나만의 인디케이터

- 팀: 4명 / 대상 시장: 다양하게 (한국·미국 주식 중심, 암호화폐 후순위)
- 목표: 기존 지표(RSI, MACD 등)를 넘어서는 ML 기반 커스텀 인디케이터 설계

---

## 후보 아이디어 10선

### A. 난이도 하 — 확실히 완성 가능

**1. ML 앙상블 시그널 스코어 (Composite Signal Score)**
- 개념: RSI, MACD, 볼린저, 거래량 등 10~20개 기술지표를 피처로 넣고, N일 후 상승 확률을 0~100 점수로 출력하는 지표
- 모델: XGBoost / LightGBM 분류
- 데이터: FinanceDataReader(한국) + yfinance(미국)
- 장점: 구현 확실, 피처 중요도 분석으로 발표 스토리가 좋음
- 리스크: 흔한 주제라 차별화 필요

**2. 변동성 국면(Regime) 인디케이터**
- 개념: 시장이 "상승추세 / 하락추세 / 횡보 / 고변동" 중 어느 국면인지 실시간 분류하는 신호등 지표
- 모델: HMM(은닉마르코프) 또는 K-Means/GMM 클러스터링
- 장점: 비지도학습이라 레이블링 불필요, 시각화가 직관적 (차트 배경색으로 국면 표시)
- 차별점: 국면별로 "어떤 전략이 유효한지"까지 연결하면 완성도 높음

**3. 이상 거래량 탐지 지표 (Volume Anomaly Detector)**
- 개념: 평소와 다른 비정상 거래량/가격 패턴을 탐지해 세력 유입·급변동 전조를 점수화
- 모델: Isolation Forest / Autoencoder
- 장점: 이상탐지는 발표 임팩트가 큼, 국내 테마주에 적용하면 재미있는 사례 다수

### B. 난이도 중 — 차별화와 완성 가능성의 균형 ★추천 구간

**4. 뉴스/공시 감성 인디케이터 (Sentiment Index)**
- 개념: 종목별 뉴스 헤드라인을 수집해 감성 점수를 시계열 지표로 만들고, 가격 지표와 결합
- 모델: KoBERT/KoELECTRA 파인튜닝 또는 사전학습 감성모델 + 가중 이동평균
- 데이터: 네이버 금융 뉴스 크롤링, 미국은 FinBERT + 뉴스 API
- 장점: "AI" 느낌이 강하고 기존 기술지표와 확실히 다름
- 리스크: 크롤링/데이터 수집 공수, 4인이면 1명 전담 필요

**5. LSTM/Transformer 기반 추세강도 지표**
- 개념: 딥러닝이 예측한 단기 방향 확률을 -100~+100 오실레이터로 변환 ("AI-RSI")
- 모델: LSTM 또는 소형 Transformer
- 장점: 기존 오실레이터와 직접 비교 가능 → RSI vs AI-RSI 백테스트 비교가 발표 킬러 콘텐츠
- 리스크: 과적합 관리, 학습 시간

**6. 캔들 패턴 인식 지표 (CNN Chart Pattern Detector)**
- 개념: 차트 이미지를 CNN으로 학습시켜 헤드앤숄더, 쌍바닥 등 패턴 발생 확률을 지표화
- 모델: CNN (이미지 분류) 또는 시계열 CNN
- 장점: 시각적으로 가장 재미있는 발표, 데모 임팩트 큼
- 리스크: 패턴 레이블링 데이터 만들기가 관건

**7. 상관관계 붕괴 지표 (Correlation Breakdown Index)**
- 개념: 자산 간(환율·금리·업종지수 등) 평소 상관구조가 무너지는 순간을 탐지 → 시장 스트레스 조기경보
- 모델: 롤링 상관행렬 + PCA/오토인코더 재구성 오차
- 장점: "다양한 시장" 요구에 가장 부합 (주식+환율+금리+원자재 통합)
- 차별점: 매크로 관점이라 다른 조와 안 겹칠 확률 높음

### C. 난이도 상 — 도전적, 시간 리스크 있음

**8. 강화학습 포지션 신호 지표**
- 개념: RL 에이전트의 행동(매수/매도/관망) 확률을 지표로 노출
- 모델: DQN / PPO
- 리스크: 학습 불안정, 기간 내 완성 난이도 높음

**9. 멀티모달 지표 (가격 + 뉴스 + 수급 통합)**
- 개념: 4·5번을 합친 통합 스코어. 외국인/기관 수급 데이터(pykrx)까지 결합
- 장점: 4인 팀 분업에 최적 (데이터별 1인씩)
- 리스크: 통합 단계에서 시간 부족 가능

**10. 유사 국면 검색 지표 (Chart Similarity / kNN)**
- 개념: 현재 차트와 가장 유사한 과거 구간을 임베딩 검색으로 찾아 "과거 이런 상황에서 N일 후 평균 수익률"을 지표로 표시
- 모델: DTW / 시계열 임베딩 + kNN
- 장점: 설명 가능성이 뛰어나 발표 설득력 높음
