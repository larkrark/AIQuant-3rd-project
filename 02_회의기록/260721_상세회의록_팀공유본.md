## 1. 지역 목표비중과 세부조정 (D-20260721-10, 전원 동의 확정)

결정질문 "지역비중은 무엇을 기준으로 정하는가"에 대해 세 안을 비교했다: 안 A 사전 고정비중 / 안 B 최종 구성종목 수 연동 / 안 C 유동시가총액 연동. **안 A 채택.** 근거는 종목 수(9:9)가 아니라 테마 1:1:1과 같은 동등 측정 원리이며, 안 B는 "지역비중은 종목 수와 별도 결정"(D-09)과 충돌하여 기각, 안 C의 시총 연동은 지역 단계가 아닌 안건 H의 종목가중 논의로 이관했다.

세부조정 확정 사항: ① 고정비중 한국 50 : 미국 50 ② 목표비중은 테마×지역 6셀 각 1/6로 정의 ③ 셀 적격종목 부족 시 같은 테마의 타지역으로 재배분(테마 1:1:1 우선), 이탈은 사유코드와 함께 기록·공시 ④ 공식 합성 BM은 지수의 정기변경 목표 지역비중과 동일 연동 — 부족규칙이 산출한 예외 목표비중(예: 45:55) 발생 시 같은 정기변경일부터 BM에 동일 적용(재량 승인이 아닌 규칙 산출값). 의도하지 않은 지역배분 효과가 알파로 읽히는 왜곡을 방지 ⑤ 합성 BM은 지수와 동일 기준일·기준값 1,000으로 리베이스 ⑥ QA 구조 구분 — 데이터·판정 QA(입력계약·누락중복·상태값·추적성·회귀·자동 최종판정 비활성)와 산출 QA(재현성·정합성·BM 연동 일치)를 나누고, 예외 분기의 정상상태(50:50) BM 대비 수익률 차이 병기는 QA가 아닌 성과분석·BM 비교 항목으로 배치.

## 2. 종목선정 규칙 체크리스트 작성과 진행 방식

조원 의견("종목선정 규칙이 가장 시급")을 받아, 유니버스→최종 구성종목까지의 선정 규칙 전체를 30개 항목·6단계로 정리한 체크리스트를 작성하고 각 항목에 데이터 필요 여부를 표시했다. A-1·A-2에서 확립한 2단계 방식(구조는 지금 표결, 숫자는 자료 확보 후 확정)을 전 항목에 적용하기로 했다. 각 규칙의 의미와 필요성 해설을 팀에 공유했다.

## 3. 단계별 취합 결과 (D-20260721-11, 항목별 상태표로 기록)

SPAC 합병 전환 재적격, 과거 유니버스 원출처(KRX+KIND·DART / 거래소+EDGAR, 소급 금지), 거래정지 사유 기반 상태 규칙 확정. 잔여 확인: NYSE American 즉시 포함(김민호), 대표 클래스 기준 문구(김근형).

**2단계 테마 게이트(#7~13)**: 3축 분리(사업역할/정량 노출상태/최종 출력) 재확인. 사업역할 5단계→출력 3단계 매핑표 확정. 근거 비대칭 원칙(A·B등급 공식근거로 역할판정까지 허용, 정량 매출비중 근거로 확대 금지) 채택. 2인 비공개 독립판정(동일 자료·규칙버전)을 절차규칙으로 룰북에 기록하고 실명은 역할배정표로 분리. 테마명 표기 혼동("우주/항공")은 확인 결과 **방산 포함 유지·표기 "우주·방산" 통일**로 정리(범위 변경 아님).

## 4. 문서·협업 체계

GitHub 저장소(공개)에 Pages 위키(Docsify, 무료·빌드 없음)를 구성했다. 전 docx 문서를 md로 변환해 웹에서 열람 가능하게 했고, 사이드바·전문 검색을 갖췄다. 문서 수정은 GitHub 웹에서 md 편집→커밋으로 즉시 반영된다(docx 수정 시 동명 md 동반 갱신 규칙). 결정로그를 단일 원본으로 재확인하고, 상세회의록 D-번호·룰북 M-번호와의 ID 대응표를 결정로그에 두었다.
