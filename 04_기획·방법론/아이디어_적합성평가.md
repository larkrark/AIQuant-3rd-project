# 아이디어별 인디케이터 적합성 평가

평가 기준: 인디케이터란 **"수치화된 시그널을 시계열로 계속 산출"**할 수 있어야 함 → ①지표화 가능성 ②데이터 확보 ③백테스트 가능성 ④ML 적용성 ⑤차별성

## 아이디어 1: "선반영" 인디케이터 (본인)

**개념 해석**: 재료(실적, 정책, 뉴스)가 발표되기 전에 가격에 얼마나 미리 반영됐는지를 수치화 → "이미 다 반영된 재료인가(sell the news), 아직 덜 반영됐는가"를 판단하는 지표

### 평가: ★ 적합성 높음 — 인디케이터 형태에 정확히 부합

| 기준 | 평가 | 근거 |
|---|---|---|
| 지표화 가능성 | 상 | 이벤트 전 누적 초과수익률, 뉴스량 대비 가격 반응 등으로 0~100 "선반영 점수" 산출 가능 |
| 데이터 확보 | 중 | 가격·거래량(쉬움) + 이벤트 일정(실적발표일·공시, 확보 가능) + 뉴스량(크롤링 필요) |
| 백테스트 | 상 | 이벤트 스터디로 검증 명확: "선반영 점수 높음 → 발표 후 하락(재료소멸)" 가설 검정 |
| ML 적용성 | 상 | 뉴스 감성·거래량·수급 피처로 발표 후 방향 예측 → 분류 모델 그대로 적용 |
| 차별성 | 상 | 기존 기술지표에 없는 관점. "소문에 사서 뉴스에 팔아라"의 정량화라는 발표 스토리 강력 |

**구현 스케치**: 선반영 점수 = f(이벤트 전 N일 초과수익률, 거래량 급증도, 뉴스 언급량 증가율, 감성 점수) → 임계값 이상이면 "재료소멸 경고"

**리스크**: "선반영"의 조작적 정의를 초반에 명확히 해야 함. 이벤트 데이터셋(실적발표일 등) 정리에 공수 소요

## 아이디어 2: 환경 테마 (조원)

해양환경보호 / 온난화(작물재배지 북상, 관개·수자원) / 탄소포집 촉매 / 한국 수질개선

**개념 해석**: 이것은 **투자 테마(종목 유니버스)**이지 그 자체로는 인디케이터가 아님. "무엇을 살까"의 답이지 "언제/얼마나 강한 신호인가"의 답이 아님

### 평가: 원형 그대로는 적합성 낮음 — 단, 지표화 가공 시 가능

| 기준 | 평가 | 근거 |
|---|---|---|
| 지표화 가능성 | 하→중 | 테마 자체는 지표가 아님. "테마 과열도 지수" 등으로 변환해야 지표가 됨 |
| 데이터 확보 | 하 | 기후·조위·작황 데이터는 월/연 단위라 매매신호 주기와 불일치. 수질개선 테마주는 소형주가 많아 데이터 얕음 |
| 백테스트 | 하 | 온난화 효과는 수십 년 단위 → 프로젝트 기간 내 검증 가능한 레이블 만들기 어려움 |
| ML 적용성 | 중 | 테마 뉴스 분류(NLP)에는 적용 가능 |
| 차별성 | 상 | 주제 자체는 참신하고 사회적 의미 있음 |

**살리는 방법**: "환경테마 과열도(Heat) 지수" — 테마 종목 바스켓의 뉴스 언급량, 상대강도, 거래대금 집중도를 결합해 테마의 자금 유입 국면을 점수화

## 아이디어 3: 우주개발 · AI시스템 · 에너지인프라 테마 + "간결하게" (조원 2)

**개념 해석**: ①유기적으로 연결된 3개 성장 테마(우주—크라토스, AI—아리스타·인튜이티브서지컬, 에너지—GE버노바·이튼)를 유니버스로 ②설계는 단순하게, 검토·발표 준비 시간 확보 ③선반영 아이디어는 부분 적용

### 평가: 테마로서는 조원 1안보다 우수, 단순화 제안은 타당

| 기준 | 평가 | 근거 |
|---|---|---|
| 지표화 가능성 | 하→상 | 테마 자체는 역시 유니버스. 단, 3개 산업의 **연결 구조**(수주→인프라→수요) 덕분에 lead-lag·상대강도 지표로 가공하기 좋음 |
| 데이터 확보 | 상 | 미국 중대형주 → yfinance로 즉시 확보, 유동성·실적발표일 데이터 모두 깨끗함. 환경 테마(소형주·기후데이터)의 약점이 전부 해소됨 |
| 백테스트 | 상 | 상장 이력 충분, 실적 이벤트 많음 |
| ML 적용성 | 중 | 단순화 방침과 결합 시 XGBoost 수준이 적정선 |
| 차별성 | 중 | 테마 자체는 시장에서 이미 핫함 → 차별화는 시그널 설계에서 확보해야 함 |

**"간결하게" 제안에 대한 판단**: 타당함. 특히 선반영 지표에서 공수가 가장 큰 부분이 **뉴스 크롤링·감성분석**인데, 이를 빼고 가격·거래량·실적발표일만으로 선반영 점수를 만들면 난이도가 "중→하"로 내려감. 창의성은 아이디어(선반영 관점)에서 이미 확보되므로 구현이 단순해도 발표 스토리는 유지됨

## 최종 결론 및 제안 (3개 의견 통합)

각 의견의 역할이 다름: **아이디어 1 = 시그널 로직**, **아이디어 2·3 = 유니버스**, **조원 2의 단순화 = 프로젝트 운영 원칙**

유니버스 비교: 조원 2안(우주·AI·에너지)이 조원 1안(환경)보다 데이터·유동성·이벤트 빈도 모두 우위 → 구현 관점에서는 조원 2안 채택이 안전. 환경 테마는 4번째 바스켓으로 1개 추가하면 조원 1 의견도 반영 가능

### ★ 통합안: "테마 선반영 라이트(Lite) 인디케이터"

> 우주·AI·에너지(+환경) 테마 바스켓에 대해, **가격·거래량만으로** 실적발표 전 선반영 점수를 산출
> 선반영 점수 = f(발표 전 N일 초과수익률, 거래량 급증도, 테마 내 상대강도)
> → 발표 후 재료소멸/추가상승 여부를 이벤트 스터디로 검증. 여유가 생기면 뉴스 감성을 후반에 추가(선택)

**단순화 포인트 (조원 2 의견 반영)**
- 뉴스 크롤링 제외 → 데이터는 yfinance 하나로 통일
- 모델은 XGBoost 1개(또는 규칙 기반 점수 → ML은 비교용)
- 종목 수 테마당 5~8개로 제한
- 마지막 2주는 구현 동결, 검토·발표 준비 전용

| 구성 | 반영된 의견 |
|---|---|
| 유니버스: 우주·AI·에너지 (+환경 1바스켓) | 조원 2 (+조원 1) |
| 시그널: 선반영 점수 (가격·거래량 기반 라이트 버전) | 본인 |
| 운영: 단순 구현 + 검토 기간 확보 | 조원 2 |
| 검증: 실적발표 이벤트 스터디 | 공통 |

4명 전원의 의견이 구조적으로 반영되고, 난이도는 "하~중"으로 기간 내 완성 가능
