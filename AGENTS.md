# AGENTS.md — AI 코드/문서 작업 규칙 (룰북 강제 반영본)

> 이 파일은 이 저장소에서 **코드·데이터·문서를 작성하거나 수정하는 모든 AI 도구**
> (Claude Cowork, Codex, Copilot 등)가 **매 작업 시작 전에 반드시 먼저 읽고 끝까지 준수**해야 하는 규칙이다.
> 파일 하나가 길어져도 아래 규칙을 "잊지" 말 것. 작업을 끝내기 전 **§E 자가 점검**을 반드시 수행한다.
>
> 근거 문서(단일 원본): `01_운영문서/260720_커스텀인덱스_구성및산출_룰북_v0.3_확정명세반영본.md`(이하 룰북),
> `01_운영문서/결정로그.md`, `01_운영문서/데이터사전.md`, `06_코드/engine/config.py`.
> 규칙과 근거 문서가 충돌하면 **근거 문서가 우선**하며, 임의로 해석하지 말고 사람에게 확인 질문을 남긴다.

## 이 프로젝트가 하는 일 (한 줄)
공개·이용가능 자료와 사전 규칙만으로 한·미 3개 테마(우주·방산 / AI·로보틱스 / 에너지·전력) 지수의
구성종목·비중·일별 지수값을 **같은 입력에서 같은 결과가 나오도록** 반복 산출한다.
목적은 알파 극대화가 아니라 **투명성·재현성·추종가능성**이다.

## §A. 절대 규칙 (하드 인바리언트 — 위반 시 코드 폐기)

각 규칙은 지켰는지 O/X로 판정 가능해야 한다. 애매하면 지키지 않은 것으로 본다.

- **R1 · 숫자 하드코딩 금지.** 모든 임계값·비중·기간·기준값·달력값은 `config.py`에만 정의한다.
  다른 모듈(`indicators.py`, `composition.py`, `index_calc.py` 등)에 리터럴 숫자(90, 0.5, 1000, 1/6 …)를 쓰지 않는다.
  *근거: config.py 헤더 / 룰북 §0*

- **R2 · 미결 항목을 확정처럼 쓰지 않는다.** 룰북에서 `미결안`·`[미결]`·`TEMPORARY`·`PENDING`인 값은
  임의의 숫자로 채우지 않는다. 잠정값은 `WEIGHTING_STATUS="TEMPORARY"`,
  `TEAM_CONFIRMATION_PENDING`처럼 상태를 코드/주석에 명시해 유지한다.
  *근거: 룰북 §0·§17 / config.py*

- **R3 · 성과 기반 규칙 조정 금지.** 산출 결과(성과·순위)를 본 뒤 `config.py`·규칙·경계값을 바꾸지 않는다.
  값 변경은 반드시 **결정로그 기록을 거친 뒤**에만 한다.
  *근거: 룰북 §0·§11 / engine/README.md*

- **R4 · PIT 준수 / look-ahead 금지.** 선정일·자료마감일(`data_cutoff_date = 선정일 − 5거래일`) **이전에
  이용 가능했던 자료만** 사용한다. `data_available_date = 공시 다음 거래일`. 정정 공시의 **소급 반영 금지**.
  미래 가격·미래 재무·당시 존재하지 않던 종목을 참조하지 않는다(생존편향 포함).
  *근거: 룰북 §5·§14 / 체크리스트 #29·#30*

- **R5 · 수정주가 오용 금지.** 거래대금(ADTV90)은 **원종가 × 원거래량**으로 계산한다.
  수정주가(adjusted price)는 **수익률 계산 전용**이며 거래대금·유동성 판정에 쓰지 않는다.
  *근거: 룰북 §8.1*

- **R6 · NA ≠ 0.** `DATA_MISSING`은 0으로 채우지 않고 **NA로 유지**하며 분모에서 제외한다(분모 = 90 − NA일수).
  `ZERO_VOLUME`(실제 무거래)·`TRADING_HALT`(거래정지)는 거래대금 **0으로 반영**한다. 이 둘을 섞지 않는다.
  *근거: 룰북 §8.1*

- **R7 · 3축 분리.** 하나의 단계/함수가 테마 관련성·유동성·재무상태·가격상태를 **동시에** 판정하지 않는다.
  적격성 게이트와 인디케이터, 사업역할과 매출비중은 별도 축으로 분리한다.
  *근거: 룰북 §4·§7.2·§8*

- **R8 · 표준 날짜 5필드만 사용.** `fiscal_date · filing_date · data_available_date(observation) · selection_date · effective_date`.
  데이터사전에 정의된 필드명을 쓰고 임의 명칭을 만들지 않는다. 가격·재무·공시·환율 기준일을 서로 다른 열로 관리한다.
  *근거: 룰북 §5.2 / 데이터사전.md*

- **R9 · 시장 상태코드 6종 고정.** `TRADED / ZERO_VOLUME / TRADING_HALT / MARKET_CLOSED / DATA_MISSING / NOT_LISTED`
  외의 코드를 만들지 않는다. **원천 상태와 계산 반영값을 분리 기록**한다.
  `TRADING_HALT`는 **종일 정지만** 해당하고 `ZERO_VOLUME`보다 우선한다.
  *근거: config.py / 룰북 §8.1 / 체크리스트 #15*

- **R10 · 가중 총합 = 1.0 검산 필수.** 셀 배정·빈 셀 재배분 후 비중 합계가 1.0(허용오차 내)인지 검산하고,
  통과하지 못하면 산출을 중단한다. 6셀 목표비중은 `CELL_TARGET_WEIGHT = 1/6`.
  *근거: 룰북 §14 / engine/README.md / config.py*

- **R11 · 테마·지역 구조 고정.** 테마 간 1:1:1, 3테마 × 2지역 = 6셀. 테마/지역 순서는 `config.THEMES`·`config.REGIONS`를
  단일 기준으로 따른다(코드마다 순서를 재정의하지 않는다).
  *근거: config.py / 룰북 §10*

- **R12 · 합성·샘플 데이터 인용 금지.** `make_sample.py` 등 합성 데이터로 만든 산출물은
  **어떤 문서·설명서·발표자료에도 인용하지 않는다**(파이프라인 검증 전용).
  파일럿(v0.9-pilot) 수치도 정식 백테스트 칸에 넣지 않는다.
  *근거: engine/README.md / pilot 메모*

- **R13 · AI 산출물 직접 인용 금지.** 공식 원문·수치·재무범위·회사·티커·최종 판정은 **사람이 확인**한다.
  AI 요약을 공식 근거로 직접 인용하지 않고, 조사자·교차검토자·검토일을 기록한다.
  *근거: 룰북 §15*

- **R14 · 결정의 단일 원본.** 모든 팀 결정은 `01_운영문서/결정로그.md`가 단일 원본이다.
  코드·문서 수정 전 **GitHub 최신본과 대조**한다(로컬 편집기가 구버전으로 덮어쓴 사고 3회 이력).
  *근거: README_폴더구조.md / 팀 이력*

- **R15 · 파일·폴더·네이밍 규칙.** 회의/운영 문서 파일명은 `YYMMDD_` 접두. 폴더 용도(01_운영문서 ~ 06_코드)를 지킨다.
  구버전은 삭제 대신 `05_참고·아카이브`로 이동. 팀 결정이 걸린 변경은 결정로그에 기록한다.
  *근거: README_폴더구조.md*

- **R16 · 비밀정보 커밋 금지.** `.env`·API 키·ID/PW·토큰을 커밋하거나 코드에 하드코딩하지 않는다.
  자격증명은 `.env`에서 읽고 `.gitignore` 유지. 코드·로그·산출물에 키 값을 출력하지 않는다.
  *근거: 표준 보안 / 저장소 .env·.gitignore*

## §B. 숫자·결정은 어디에 있나 (수정 전 반드시 확인)

| 무엇 | 단일 원본 |
|---|---|
| 임계값·비중·기간·기준값·달력 | `06_코드/engine/config.py` |
| 팀 결정(확정/기본안/미결 상태 포함) | `01_운영문서/결정로그.md` |
| 필드명·상태코드·계산 명세 | `01_운영문서/데이터사전.md`, 룰북 §8.1 |
| 파이프라인 구조·모듈 역할 | `06_코드/engine/README.md` |

값을 바꾸려면: ① 결정로그 확인 → ② 미결이면 사람에게 질문 → ③ 승인된 경우만 `config.py` 수정 + 결정로그 기록.

## §C. 코드 규약 (엔진)
- 숫자·정책은 `config.py`에서 import해서 쓴다(R1). 모듈 간 역할 경계(§A R7)를 넘지 않는다.
- 인코딩 `# -*- coding: utf-8 -*-`. 필드명은 데이터사전 기준(R8).
- 상태·사유코드는 값과 함께 **판정사유**를 남긴다(왜 편입/제외인지 재현 가능하게).
- `check_data_paths.py`처럼 "로컬에서만 실행" 표시가 있는 스크립트의 실행 위치를 지킨다.

## §D. 저장소 위생
- `.env`, `__pycache__/`, `*.pyc`, `~$*`, `_to_delete/`는 `.gitignore`로 계속 제외(R16).
- 대용량 입력 CSV·중간 산출물은 정본 폴더 하나만 유지하고 중복 복제를 만들지 않는다.
- 임시 인계용 폴더(`check_this/` 등)는 정식 폴더로 옮기거나 제외 대상에 넣는다.

## §E. 끝내기 전 자가 점검 (필수)
코드/데이터 산출을 만든 뒤, **생성과 검증을 분리해서** 아래를 수행한다:
1. `AI_코드작성_룰준수_자가채점_프롬프트.md`의 채점표를 그대로 작성한다
   (규칙 R1~R16 각각 O/X/NA + 근거 `파일:줄` + 위반 시 수정안).
2. **X가 하나라도 있으면 고친 뒤 다시 채점**하여 전부 O 또는 NA가 될 때까지 반복한다.
3. 자동 검산(가중합=1.0, 상태코드 집합, look-ahead 스캔 등)을 돌려 통과를 확인한다.
4. 미결·잠정 값을 확정처럼 쓴 곳이 없는지 마지막으로 확인한다(R2).
