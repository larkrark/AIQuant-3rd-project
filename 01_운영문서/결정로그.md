# 결정로그 (Decision Log)

한·미 테마형 패시브 커스텀 인덱스 프로젝트 — 팀 공통 결정 기록

> 운영 원칙 (260720 회의안건 문서 기준)
> - 성과를 확인하기 전에 규칙버전·효력발생일·결정사유를 저장한다.
> - 자료가 없는 항목은 임의의 숫자로 채우지 않고 필요한 자료와 담당자를 남긴다.
> - 검토한 대안과 기각사유를 함께 기록한다. 초기 의견의 오류 지적이 아니라 변경관리 기록이다.
> - 문서와 코드 설정의 rule_version을 일치시킨다.

---

## 상태 정의

| 상태 | 의미 |
|---|---|
| 상정 | 회의에 올라온 제안. 아직 팀 결정 전 |
| 잠정 | 기본안 채택. 비교자료 또는 팀 승인 필요 |
| 확정 | 팀 공통 기준으로 채택. 문서·코드에 동일 적용 |
| 미결 | 지수 결과에 직접 영향. 회의에서 결정 필요 |
| 보류 | 현재 제출 범위에서 후속 연구로 이동 |

## 기록 양식 (새 결정은 이 표를 복사해서 작성)

| 필드 | 내용 |
|---|---|
| 결정ID | D-YYYYMMDD-## (예: D-20260720-01) |
| 안건 연계 | 회의안건 A~K 중 해당 항목 |
| 결정 내용 | 한 문장으로. 무엇을 어떻게 하기로 했는가 |
| 상태 | 상정 / 잠정 / 확정 / 미결 / 보류 |
| 근거·사유 | 왜 이렇게 결정했는가 (원문·문서 근거 포함) |
| 검토한 대안 | 무엇을 비교했는가 |
| 기각 사유 | 대안을 왜 채택하지 않았는가 |
| 영향 범위 | 이 결정이 입력이 되는 후속 단계·문서·코드 |
| 필요 자료·후속 확인 | 결정을 확정하기 위해 남은 자료와 담당 |
| 제안자 / 승인 | 제안자, 승인 방식(전원 합의 / 담당 승인) |
| 결정일 / 효력발생일 | 결정한 날 / 규칙이 적용되는 기준일 |
| rule_version | 이 결정이 반영되는 규칙 버전 (승인 전 v0.x) |
| 관련 링크 | 회의록·조사자료·공시 원문(접수번호) 등 |
