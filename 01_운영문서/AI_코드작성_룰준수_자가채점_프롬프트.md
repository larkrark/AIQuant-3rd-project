# AI 코드 작성 — 룰북 준수 자가 채점 프롬프트

AI로 코드를 짠 **직후**, 아래 블록을 **그대로 복사해 붙여넣어라**.
가장 효과가 좋은 방법은 **새 대화(또는 새 세션)** 에서 코드를 붙여넣고 이 프롬프트를 실행하는 것이다
— 코드를 짠 맥락에 물들지 않은 상태로 채점해야 "자기 코드는 다 맞다"는 편향이 빠진다.

---

## 사용법
1. AI가 만든 코드(또는 변경 diff)를 준비한다.
2. 아래 `=== 채점 프롬프트 시작 ===` ~ `=== 끝 ===` 사이를 복사한다.
3. 새 대화에 붙여넣고, 그 아래에 **채점할 코드**를 붙인다.
4. X가 하나라도 나오면 **고친 뒤 다시 이 프롬프트로 재채점**한다. 전부 O/NA가 될 때까지 반복한다.

---

=== 채점 프롬프트 시작 ===

너는 지금 **코드 작성자가 아니라 독립 검증자**다. 아래 코드가 이 저장소의 룰북(AGENTS.md §A, R1~R16)을
지켰는지 **한 줄씩 실제 코드를 근거로** 채점하라. "아마 지켰을 것"이라고 넘어가지 말고, 근거 줄을 못 찾으면 X로 판정한다.

**출력은 아래 표 하나로만** 한다. 각 행에:
- 준수: `O`(지킴) / `X`(위반) / `NA`(이 코드와 무관)
- 근거: 판단의 근거가 된 **파일:줄번호와 실제 코드 한 조각**(O든 X든 반드시 인용). 근거를 못 대면 X.
- 위반 시 수정안: X인 경우 **어떻게 고쳐야 하는지 구체적으로**.

| 규칙 | 검사 항목 | 준수(O/X/NA) | 근거(파일:줄 + 코드) | 위반 시 수정안 |
|---|---|---|---|---|
| R1 | 임계값·비중·기간·기준값이 config.py에만 있고, 다른 모듈에 리터럴 숫자(90·0.5·1000·1/6 등) 하드코딩 없음 | | | |
| R2 | 미결·잠정 값을 임의 숫자로 확정하지 않고 TEMPORARY/PENDING 상태로 표시·유지 | | | |
| R3 | 성과·순위 결과를 본 뒤 config·경계값을 바꾸는 로직/흔적 없음 | | | |
| R4 | PIT 준수 — 선정일/자료마감일 이후 자료·미래값·당시 미존재 종목 참조 없음, 정정 소급 없음 | | | |
| R5 | 거래대금(ADTV90)은 원종가×원거래량, 수정주가는 수익률 전용 (거래대금에 수정주가 미사용) | | | |
| R6 | DATA_MISSING은 NA 유지(분모 제외), ZERO_VOLUME·TRADING_HALT만 0 반영 — 둘을 안 섞음 | | | |
| R7 | 한 함수/단계가 테마관련성·유동성·재무·가격을 동시에 판정하지 않음(축 분리) | | | |
| R8 | 표준 날짜 5필드명(fiscal/filing/available/selection/effective) 사용, 임의 필드명 없음 | | | |
| R9 | 상태코드 6종만 사용, 원천상태와 반영값 분리, TRADING_HALT=종일정지·ZERO_VOLUME보다 우선 | | | |
| R10 | 셀 배정·재배분 후 비중 합계 1.0 검산 코드 존재, 실패 시 중단 | | | |
| R11 | 테마 1:1:1·6셀·1/6 구조, 테마/지역 순서는 config.THEMES/REGIONS 기준 | | | |
| R12 | 합성/샘플 데이터 산출물을 문서·발표·정식 결과에 인용하지 않음 | | | |
| R13 | AI 산출 요약을 공식 근거로 직접 인용하지 않음, 조사자·교차검토자 기록 | | | |
| R14 | 팀 결정값을 결정로그와 대조해 사용(임의 결정 도입 없음) | | | |
| R15 | 파일·폴더·네이밍(YYMMDD, 01~06 폴더 용도) 규칙 준수 | | | |
| R16 | .env·API키·ID/PW·토큰을 코드에 하드코딩하거나 출력하지 않음 | | | |

표 아래에 다음을 덧붙여라:
1. **위반(X) 개수**와 목록.
2. X가 있으면: 수정한 전체 코드.
3. **자동 검산 제안**: 이 코드에 붙일 수 있는 최소 테스트(예: 가중합=1.0 assert, 상태코드 집합 검사,
   selection_date 이후 날짜 참조 스캔) 코드.
4. 판정이 애매해 사람 확인이 필요한 항목은 **[사람확인]** 으로 따로 표시.

규칙 원문이 필요하면 저장소 루트 `AGENTS.md`의 §A를 기준으로 삼아라.

=== 끝 ===

## 왜 이 방식이 100%에 가까워지는가
- **생성과 검증을 분리**한다 — 짠 사람(맥락)과 채점하는 사람(백지)을 나누면 "자기 합리화"가 줄어든다.
- **근거 줄번호를 강제**한다 — "지켰습니다"가 아니라 코드를 인용해야 하므로 빈말을 못 한다.
- **X=0까지 루프**한다 — 한 번 훑고 끝내지 않고 전부 O/NA가 될 때까지 돌린다.
- **자동 검산으로 마감**한다 — 사람이 눈으로 보는 대신, 가중합·상태코드·look-ahead를 기계가 잡는다.
