본 문서는 지수의 목적·범위·제약·거버넌스를 규정하는 최상위 문서이다. 세부 계산·판정 규칙은 룰북(결정로그에서 승인된 규칙)이 규정하며, 본 문서와 룰북이 충돌할 경우 본 문서가 우선한다. 각 조항의 상태는 [확정]·[잠정]·[미결]로 표시한다.

## 1. 목적 [확정]

한국과 미국 상장기업 가운데 우주·방산, AI·로보틱스, 에너지·전력 인프라와 직접 또는 핵심적으로 관련된 종목을 공식 공시와 사전 정의한 구성규칙으로 판정하고, 편입·유지·제외·가중·정기변경 규칙을 반복 적용하여 세 테마의 구성과 시장성과를 측정한다.

## 2. 지수 정체성 [확정]

- 규칙 기반 패시브 지수이다. 사전 공개된 규칙이 구성을 결정하며 재량 판단을 배제한다.
- 시장 측정 지표이며 비개인화 상품이다.
- 평가 기준은 규칙의 투명성·재현성·추종가능성이다 (액티브식 알파·신호 정교함이 아님).

## 3. 수행범위에서 배제하는 것 [확정]

- 투자자별 위험성향에 따른 자산배분을 수행하지 않는다.
- 개별 종목의 매수·매도 시점이나 가격 방향을 제시하지 않는다.
- 임계값·규칙을 성과 결과를 본 뒤 조정하지 않는다 (사전 확정 원칙).
- 자료가 없는 항목을 임의의 숫자로 채우지 않는다 (결측은 사유 코드로 기록).

## 4. 대상 유니버스

- 한국·미국 상장 보통주 [확정]. 포함 거래소: 한국 코스피 / 미국 나스닥·NYSE [잠정 — 코스닥 포함 여부 재확인 중].
- ETF·ETN·우선주·REIT·SPAC·ADR·KONEX는 제외 [잠정 — 제외코드표 확정 대기].
- 기준일별 상장상태는 point-in-time으로 기록하며 생존편향을 차단한다 [확정 — 과거 상장상태 원출처는 미결].

## 5. 테마와 배분 [확정]

- 테마: 우주·방산 / AI·로보틱스 / 에너지·전력 인프라.
- 테마 간 비중 1:1:1 (계산 시 각 1/3 정규화).
- 테마 소속은 공식 공시 원문 기반 테마 적격성 게이트가 결정한다. 게이트는 인디케이터가 아닌 별도 필수 규칙이다 (결정로그 D-01).

## 6. 구성 절차 개요 [확정된 골격]

기준일별 시장 유니버스 확정 → 공식 공시 원문 수집(PIT) → 테마 노출도 조사·기록 → 테마 적격성 게이트 → 상장 후 유효거래일 90일 시즈닝(D-06) → ADTV90 유동성 기준(D-07) → 재무건전성 확인 [미결 — 항목 확정 대기] → 편입·유지·제외 규칙 [미결] → 가중 [미결 — 테마 1:1:1 외] → 지수 산출·성과 계산 [미결 — PR/TR·환율] → MACD 상태 진단(편입·가중 불사용).

테마 적격성 통과는 최종 편입과 동일하지 않다. 게이트 통과 후에도 구성 규칙(종목 수·버퍼)을 거쳐야 편입이 확정된다.

## 7. 자료와 판정 원칙 [확정]

- 원자료는 DART·SEC 등 공식 공시를 우선하며 point-in-time으로 사용한다.
- AI 도구는 탐색·요약·양식 작성의 보조로만 사용한다. 최종 판정은 사람이 원문을 확인하고 접수번호·공시일·원문 위치·검토자를 기록한다.
- 원천 상태와 계산 반영값을 분리 기록하여, 처리방식이 변경되어도 원천에서 재계산할 수 있게 한다.

## 8. 리밸런싱 [주기 확정]

분기 단위(약 60거래일) 정기 리밸런싱. 관측일·선정일·적용일과 수시변경 세부규칙은 룰북에서 확정한다 [미결].

## 9. 벤치마크와 환율 [확정 — 2026-07-20 팀 승인(D-08), 세부는 조건부]

- 공식 BM은 "프로젝트 공식 합성 BM"으로 한다: KOSPI 200 PR과 Russell 3000 PR을 결합한 합성 시장 벤치마크. 지역가중은 지수의 지역비중과 동일하게 연동한다 [지역비중 확정 시 같이 확정].
- 지수·BM 모두 v1은 PR로 통일한다. 사유: 동일한 기간·빈도·기준으로 재현 가능한 공식 TR 시계열을 현재 프로젝트에서 안정적이고 비용 없이 확보하기 어렵다. TR은 배당 데이터·라이선스 확보 시 후속 버전에서 정식 산출하며, PR 지수와 TR BM의 혼용 비교는 금지한다.
- 기준통화는 원화(무헤지)이며, 환율은 한국은행 ECOS의 원/달러 매매기준율 계열을 지수·BM에 동일 적용한다. 정확한 통계표·계열코드, 원/달러 방향, 일별 기준시각, 한·미 휴장일 불일치 처리, 결측일 처리, 선정일·효력발생일 적용 환율은 산출규칙(J)에서 확정한다 [미결 — D-08 ③ 조건].
- 테마 ETF(ARKK·XLU 등)는 BM이 아닌 특성 비교군으로 구분한다.

## 10. 재현성 요건 [확정]

동일한 기준일·원자료·규칙 버전을 사용하면 누가 산출하더라도 같은 구성종목·지수 비중·지수값이 산출되어야 한다. 최종 제출 전 대표 과거 검토일에 대한 독립 재산출로 이를 검증한다.

## 11. 거버넌스 [확정]

- 모든 팀 결정은 결정로그에 결정ID·근거·검토 대안·기각사유·rule_version과 함께 기록한다.
- 규칙 버전은 검토본(v0.x) → 적용시험본 → 승인본 순서로 승격하며, 표본 적용·교차검토·코드 적용시험·팀 승인 전에 동결하지 않는다.
- 문서와 코드 설정의 rule_version을 일치시킨다.
- 협업: 일상 문서는 GitHub 공동 페이지에서 보강, 코드 변경은 작업 브랜치 [상정 — D-05].
