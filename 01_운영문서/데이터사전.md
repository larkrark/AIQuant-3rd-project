# 데이터 사전 v0.2 (병합수정본)

한·미 우주·AI·에너지 테마형 지수 — 필드·상태코드·산식의 단일 정의 문서

| **항목** | **내용** |
|---|---|
| 프로젝트 | 공식 공시와 정량조건을 활용한 규칙 기반 한·미 테마형 패시브 커스텀 인덱스 개발 |
| 문서 성격 | 기본 필드·기록 필드·상태코드·관계키 정의 |
| 기준시점 | 2026-07-23 (v0.1 md + v0.2-review docx 병합, 정합성 수정 4건 반영) |
| 데이터사전 버전 | `v0.2` |
| 연계 문서 | 통합 룰북 / 수식정리부록 / 결정로그(단일 원본) / 원장 템플릿 |
| 문서 상태 | 확정 필드와 시험·검토용 필드를 구분한 통합본. 위키 md가 원본이며 docx는 동반 사본 |

> 본 데이터 사전은 규칙 실행에 필요한 **기본 필드**와 조사·판정·검증 과정을 남기는 **기록 필드**를 구분한다.
> `NA`는 0이 아니며, 값이 없는 이유를 계산상태·판정상태·결측사유와 함께 기록한다.
> 규칙의 결정 이력은 결정로그가 단일 원본이며, 충돌 시 결정로그가 우선한다.

> **v0.2 병합 시 수정 4건** (결정로그 기록 대비 정합성): ① 시장상태 코드에 `NOT_LISTED` 복원(6종, D-07) ② 사업역할 5단계의 다섯째 값을 `THEME_UNRELATED`(테마 무관)로 복원(D-11 #8) ③ Boolean 저장값은 기록된 1/0 표준 유지(D-12 ⑩ — TRUE/FALSE 전환 제안은 재확정 대기) ④ `excluded_conditional_accounts` 필드 추가(#17 기재안 채택분). 상세는 문서 말미 "변경 이력·재확정 대기" 참조.

## 4.1.1 가격 열 사용 목적표 (D-07·제59조 — 2026-07-24 등재)

| 사용 목적 | 사용할 값 |
|---|---|
| 주가 수익률·지수 산출 | `adj_close` |
| 액면분할 전후 연속성 | `adj_close` |
| 한국 ADTV90 | KRX 공식 거래대금 우선 |
| 한국 공식 거래대금이 없을 때 | `raw_close` × 실제 거래량 |
| 미국 ADTV90 | `raw_close` × 실제 거래량 재구성 |
| 원자료·기업행위 검증 | `raw_close`, `adj_close` 모두 보존 |

> **adj_close 정의 (확정 대기 → 팀 [O] 시 확정)**: 분할 등 주식수 변동만 조정, 배당 미조정 — PR(D-08) 정합. 한·미 조정 기준 비대칭(야후=배당 포함) 주의.
> `adj_close` 미확보 상태에서는 엔진이 `raw_close` 폴백 + 경고를 내고 `valuation_price_source=RAW_CLOSE_FALLBACK`으로 기록한다 — 이 표는 수정주가 확보 후의 목표 상태. 검증 활용: `raw_close ÷ adj_close` 비율 변화일 = 기업행사 발생일 자동 탐지(QA 스캔 항목).

## 6.6 보수적 하한값 산출 조건과 게이트 전환 조항 (#17 확정)

보수적 하한값 $QR^{lower}$(조건부 계정 제외)은 다음 6개 조건을 **모두** 충족할 때에만 산출한다.

1. 현금 및 현금성자산, 유동 순매출채권, 유동부채를 확인할 수 있다.
2. 조건부 계정의 계정명과 금액을 식별할 수 있다.
3. 조건부 계정을 제외해도 다른 포함계정과 중복계상 문제가 발생하지 않는다.
4. 연결·별도 기준, 재무기준일, 통화와 단위가 일치한다.
5. 불확실성이 유동 금융자산 $S$의 일부 계정에만 한정된다.
6. 유동부채가 0보다 크고 정상적인 값으로 확인된다.

필수계정이나 계정범위의 정합성을 확인할 수 없으면 `quick_ratio_status = CALCULATION_HOLD`, 계산값 NA로 기록한다.

> **게이트 전환 시 조항 (원본: 본 데이터 사전)** — 향후 당좌비율을 게이트로 전환하더라도, `CALCULATED_LOWER_BOUND` 상태의 종목은 하한값 미달만을 이유로 자동 제외하지 않는다. 해당 종목은 추가확인 대상(`quick_ratio_review_required = 1`)으로 분류하여 사람 확인을 거친다.
